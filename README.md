# Управление требованиями, качеством, задачами

## Описание
В рамках проекта изучила процесс управления требованиями и качеством программного продукта на протяжении жизненного цикла разработки. Была выполнена классификация требований, определены их атрибуты, статусы и связи. Командой разработали тест-кейсы, отчеты о дефектах и познакомились с системой управления задачами для ведения требований и контроля процесса разработки - Jira.

## Полученные навыки
- Классификация требований, их приоритетизация, статусы и взаимосвязи
- Построение трассируемости требований между разными уровнями
- Разработка позитивных и негативных тест-кейсов
- Оформление отчетов о дефектах (Bug Report) и анализ результатов тестирования
- Основы работы с системой управления требованиями и задачами (Jira).

## Этапы проекта
1. Классификация требований и определение их атрибутов, статусов и идентификаторов
2. Регистрация требований, установление связи и зависимости между ними
3. Разработка позитивных и негативных тест-кейсов для пользовательских сценариев
4. Моделирование процесса тестирования и подготовка отчетов о найденных дефектах
5. Использование системы управления задачами для ведения требований и отслеживания их реализации

## Заключение
Проект позволил освоить управление требованиями на всех этапах разработки, получить практический опыт подготовки тестовой документации и регистрации дефектов, а также познакомиться с инструментами управления задачами и контроля качества программного продукта.
